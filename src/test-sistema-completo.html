📚 Pasos para Resolver el Error 500

1

Reemplaza el archivo PHP

Descarga cotizar-v2-DEBUG.php y súbelo a tu servidor en:

https://inedito.digital/api/envios/cotizar.php
⚠️ Asegúrate de que la ruta sea exactamente esa
2

Verifica que el archivo esté activo

Abre esta URL en tu navegador:

https://inedito.digital/api/envios/cotizar.php

Debe mostrar:

{"success":false,"message":"Método no permitido. Use POST."}
✅ Si ves ese mensaje, el archivo existe y funciona
3

Revisa los logs del servidor

En cPanel → Metrics → Errors, busca:

=== INICIO cotizar.php ===
📥 Procesando petición POST...
❌ EXCEPTION: ...

El mensaje de excepción te dirá exactamente qué está fallando.

4

Usa datos temporales mientras tanto

Sube cotizar-TEMPORAL-MOCK.php y cambia temporalmente la URL en tu código:

const ENVIOS_BACKEND_URL = 
  "https://inedito.digital/api/envios/cotizar-temporal.php";

Esto te permitirá seguir desarrollando con datos de prueba.

💡 Soluciones a Problemas Comunes

❌ Error 500: Internal Server Error

Causas posibles:

  • Error de sintaxis en el PHP
  • CURL no está habilitado
  • Permisos incorrectos del archivo (debe ser 644)
  • Límites de memoria PHP excedidos

Solución:

  1. Usa el archivo cotizar-v2-DEBUG.php (tiene mejor manejo de errores)
  2. Revisa los logs en cPanel
  3. Verifica que CURL esté instalado: php -m | grep curl
❌ Error: Failed to fetch

Causas posibles:

  • El archivo PHP no existe en la ruta
  • Problema de CORS
  • Firewall bloqueando la conexión

Solución:

  1. Verifica que la URL sea exactamente: https://inedito.digital/api/envios/cotizar.php
  2. Abre la URL en el navegador - debe mostrar el mensaje de "Método no permitido"
  3. Verifica los headers CORS en el archivo PHP
✅ Solución Temporal Garantizada

Mientras resuelves la API real, usa cotizar-TEMPORAL-MOCK.php:

  1. Sube el archivo a tu servidor como cotizar-temporal.php
  2. En tu código, cambia temporalmente:
    const ENVIOS_BACKEND_URL = "https://inedito.digital/api/envios/cotizar-temporal.php";
  3. Este archivo retorna datos de prueba pero realistas
  4. Funciona 100% sin necesidad de la API externa
📞 Necesitas ayuda de la API?

Si después de estos pasos el problema persiste, puede que las credenciales o la URL de la API sean incorrectas. Contacta a soporte de enviosinternacionales.com con:

  • Tu API Key
  • El error específico que recibes
  • Los logs de tu servidor
